# HH.ru Job Parser & Analyzer (Async + MongoDB)

Персональный инструмент для автоматизированного поиска, сбора и анализа вакансий с HeadHunter через открытый API. Никакой лишней рекламы, OAuth-авторизации и ручного скроллинга.

## 🚀 Основные возможности

* **Асинхронный сбор данных:** Использование `aiohttp` и `asyncio` позволяет скачивать сотни вакансий за считанные секунды.
* **NoSQL хранилище:** Данные сохраняются в **MongoDB**, что исключает дубликаты (по уникальному индексу `hh_id`) и позволяет хранить неструктурированные описания.
* **Умная фильтрация:** Поиск по ключевым словам (логика AND/OR), опыту, графику (удаленка) и регионам.
* **Генерация отчетов:** Автоматическое создание сводки в формате **Markdown** с аналитикой навыков, городов и зарплат.

## 🛠 Стек технологий

* **Python 3.12+**
* **MongoDB** (база данных)
* **Motor** (асинхронный драйвер для MongoDB)
* **aiohttp** (асинхронные HTTP-запросы)

## 📦 Установка и застройка (Ubuntu 24.04)

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-username/hh-parser.git
   cd hh-parser
   ```

2. **Настройте виртуальное окружение:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install aiohttp motor
   ```

3. **Убедитесь, что MongoDB запущена:**
   Скрипт по умолчанию ожидает подключение по адресу `mongodb://localhost:27017`.

## 🖥 Использование

### 1. Сбор вакансий (`main.py`)
Скрипт подключается к API HH, ищет вакансии по заданным фильтрам и сохраняет новые записи в базу данных.
```bash
python3 main.py
```
*Настройки поиска (ключевые слова, опыт) меняются внутри файла в блоке настроек.*

### 2. Генерация отчета (`report_script.py`)
Извлекает накопленные данные из MongoDB и формирует файл `vacancy_report.md`.
```bash
python3 report_script.py
```

## 🤖 Автоматизация (Cron)

Чтобы каждое утро получать свежую выборку, добавьте задачу в `crontab -e`:
```bash
# Запуск парсинга и генерации отчета каждый день в 09:00
0 9 * * * /home/user/hh_parser/venv/bin/python3 /home/user/hh_parser/main.py && /home/user/hh_parser/venv/bin/python3 /home/user/hh_parser/report_script.py
```

## ⚠️ Важные замечания

* **User-Agent:** Всегда указывайте уникальный заголовок в настройках, иначе API HeadHunter вернет ошибку 400 (Bad Request).
* **Rate Limits:** Не устанавливайте слишком высокую частоту запросов. Асинхронный режим очень быстр, и при чрезмерной активности ваш IP может быть временно заблокирован.
